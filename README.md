# 🔐 Безопасная серверная инфраструктура

> Полный цикл: аудит, укрепление, документация, передача владельцу

## О проекте

Проект выполнен в роли **DevSecOps / Infrastructure Engineer**.
Два VPS-сервера настроены для работы VPN с каскадной маршрутизацией трафика (RU → NL).
Проведён полный аудит безопасности, устранены векторы атак, панели управления скрыты за SSH-туннелями.
Подготовлен комплект документации из 18 файлов для передачи инфраструктуры владельцу.

## 🛠 Стек технологий

| Категория | Технологии |
|-----------|------------|
| **OS** | Ubuntu Server 22.04 / 24.04 |
| **VPN/Прокси** | Xray, VLESS + TCP Reality |
| **Панель управления** | 3x-ui v3.5.0 |
| **Веб-сервер** | Nginx, Let's Encrypt |
| **Туннелирование** | SSH Tunneling, Reverse Proxy |
| **Файрвол** | UFW, iptables |
| **Мониторинг** | Netdata, systemd, journalctl, ss, netstat |
| **Бэкапы** | SQLite dump, конфиги, shell-скрипты |
| **Аутентификация** | SSH-ключи Ed25519, passphrase |

## 📋 Что сделано

### Аудит и безопасность
- [x] Read-only аудит двух серверов
- [x] Отключена парольная аутентификация SSH
- [x] Настроен вход только по SSH-ключам
- [x] Панели 3x-ui привязаны к localhost (127.0.0.1)
- [x] Доступ к панелям только через SSH-туннель
- [x] Скрыты webBasePath панелей (нестандартные пути)
- [x] Подписка вынесена на нестандартный URI
- [x] Публичный порт подписки заменён на Reverse Proxy
- [x] Настроен HTTPS для подписки (Nginx + Let's Encrypt)
- [x] Проверена целостность баз данных (SQLite integrity)
- [x] Настроен мониторинг сервера (Netdata)
- [x] Восстановление SSH после критической ошибки конфигурации

## 🏗 Архитектура

```
Клиент
  │
  ▼
┌─────────────────────┐
│   RU-сервер          │
│   VLESS TCP Reality  │
│   Порт 443           │
│                     │
│   RU-трафик → direct │
│   World → NL-сервер  │
└─────────┬───────────┘
          │ каскад
          ▼
┌─────────────────────┐
│   NL-сервер          │
│   VLESS TCP Reality  │
│   Порт 443           │
│                     │
│   direct/freedom     │
└─────────────────────┘

Публичный сайт (Nginx):
┌──────────────────────┐
│ HTTPS :2096          │
│ └─ /sub/* → :2097    │  (внутренний сервис)
└──────────────────────┘

Панели управления (только SSH-туннель):
┌──────────────────────┐
│ 127.0.0.1:23432 (RU) │
│ 127.0.0.1:9843  (NL) │
└──────────────────────┘
```

## 📊 Результаты

| Метрика | Было | Стало |
|---------|------|-------|
| Открытых портов | 4+ | 2 (443 + 2096) |
| Доступ к панели | По паролю извне | Только SSH-туннель |
| Безопасность подписки | HTTP :2097/sub/ | HTTPS :2096/уникальный-путь |
| Аутентификация SSH | Пароль | Только ключ + passphrase |
| Документация | Отсутствовала | 18 файлов + read-only скрипт |
