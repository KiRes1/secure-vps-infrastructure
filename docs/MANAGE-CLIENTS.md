# 🔧 Управление клиентами

Продление, блокировка, изменение лимитов и удаление пользователей.

## Все операции выполняются на RU-сервере

---

## 1. Включение и отключение клиента

### Отключить (временно заблокировать):

1. Зайти в панель → **Входящие** → выбрать inbound (порт 443)
2. Найти клиента в списке
3. Снять галочку **Включён** (Enable) → Сохранить

Клиент перестанет подключаться, но его конфигурация сохранится.

### Включить обратно:

1. Найти клиента
2. Поставить галочку **Включён** → Сохранить

---

## 2. Продление клиента

### Изменить срок действия:

1. Найти клиента → нажать на него
2. Поле **Срок действия** (Expiry Date):
   - Поставить новую дату
   - Или `0` — бессрочно
3. Сохранить

### Автопродление:

Поле **Автопродление (дней)**: если указать число дней, после истечения срока он автоматически продлится на это количество дней.

---

## 3. Изменение лимитов

### Лимит трафика:

1. Поле **Лимит трафика (GB)**
2. Указать новый лимит в гигабайтах
3. `0` = безлимит
4. Сохранить

### Лимит IP (количество устройств):

1. Поле **Лимит IP**
2. Указать максимальное количество одновременных подключений
3. `0` = без ограничений
4. Сохранить

---

## 4. Сброс использованного трафика

1. Найти клиента
2. Нажать кнопку **Сбросить трафик** (Reset Traffic)
3. Подтвердить

Счётчик трафика обнулится, лимит начнёт отсчитываться заново.

---

## 5. Удаление клиента

1. Найти клиента в списке
2. Нажать кнопку **Удалить** (Delete) или значок корзины
3. Подтвердить удаление

⚠️ **Внимание:** удаление необратимо. Клиент потеряет доступ.  
Для временной блокировки используйте отключение (п.1).

---

## 6. Отзыв доступа (смена UUID)

Если нужно гарантированно отозвать доступ (например, клиент передал конфиг третьим лицам):

1. Найти клиента → нажать на него
2. Сгенерировать **новый UUID** (кнопка Generate)
3. При необходимости сгенерировать **новый SubId**
4. Сохранить

⚠️ Старая ссылка перестанет работать. Нужно выдать клиенту новую.

---

## 7. Проверка активности клиента

### В панели:

- Зелёная точка — **в сети** (подключён сейчас)
- Серая точка — **не в сети**
- Поле **Трафик** — использовано/лимит
- Поле **Остаток** — сколько осталось трафика/дней

### Статистика:

В разделе **Клиенты** можно увидеть:
- Кто онлайн
- Кто исчерпал трафик
- У кого истекает срок
- Скорость передачи данных

---

## 8. Проверка логов IP

Если включено логирование IP:

1. Найти клиента
2. Поле **Лог IP-адресов** — показывает IP, с которых подключался клиент
3. Можно использовать для проверки, не передал ли клиент доступ другим

---

## 9. Массовые операции

В панели 3x-ui можно:

- **Сбросить трафик** всем клиентам inbound
- **Экспортировать** список клиентов
- **Импортировать** клиентов из файла

---

## 10. Памятка

| Действие | Где | Последствия |
|----------|-----|-------------|
| Отключить | Снять Enable | Клиент не может подключиться, конфиг сохранён |
| Включить | Поставить Enable | Клиент снова может подключиться |
| Сменить UUID | Новый UUID | Старая ссылка перестаёт работать |
| Удалить | Delete | Полное удаление, необратимо |
| Сбросить трафик | Reset Traffic | Счётчик трафика обнуляется |
| Изменить лимит | Поле Лимит трафика | Новый лимит применяется сразу |
| Изменить срок | Поле Срок действия | Новый срок применяется сразу |

---

## 11. Нельзя делать

- ❌ Менять настройки inbound (порт, Reality keys) без перевыпуска всех клиентов
- ❌ Удалять inbound
- ❌ Создавать клиентов на NL-сервере
- ❌ Менять routing без проверки
- ❌ Копировать UUID между клиентами
