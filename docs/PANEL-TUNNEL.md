# 🔒 Доступ к панелям через SSH-туннель

Скрытие административных панелей 3x-ui от публичного доступа.

## Зачем это нужно

По умолчанию панель 3x-ui доступна из интернета по IP:порт. Это создаёт риски:
- Брутфорс пароля
- Сканирование портов
- Эксплуатация уязвимостей веб-интерфейса

Решение: привязать панель к локальному интерфейсу (127.0.0.1) и заходить через зашифрованный SSH-туннель.

---

## 1. Привязка панели к localhost

### На сервере:

```bash
/usr/local/x-ui/x-ui setting -listenIP 127.0.0.1
systemctl restart x-ui
```

### Проверка:

```bash
ss -lntp | grep <порт_панели>
```

Правильный вывод:
```
LISTEN 0 4096 127.0.0.1:<порт> ...
```

Неправильный вывод (всё ещё доступна извне):
```
LISTEN 0 4096 0.0.0.0:<порт> ...
LISTEN 0 4096 *:<порт> ...
```

---

## 2. Запуск SSH-туннеля

### На Windows (PowerShell):

```powershell
ssh -N -T -L <локальный_порт>:127.0.0.1:<порт_панели> root@<IP>
```

**Разбор параметров:**

| Параметр | Значение |
|----------|----------|
| `-N` | Не выполнять команды на сервере (только туннель) |
| `-T` | Не выделять терминал |
| `-L` | Проброс порта: локальный_порт:удалённый_адрес:удалённый_порт |

**Пример:**

```powershell
ssh -N -T -L 33333:127.0.0.1:23432 root@<IP>
```

Окно PowerShell не закрывать — пока оно открыто, туннель работает.

---

## 3. Открытие панели в браузере

После запуска туннеля панель доступна по адресу:

```
https://127.0.0.1:<локальный_порт>/<webBasePath>/
```

**Пример:**

```
https://127.0.0.1:33333/xawE6594uzhpKBATfk/
```

### Предупреждение о сертификате

Браузер покажет предупреждение «Ваше подключение не защищено». Это нормально — сертификат самоподписанный или выпущен на IP сервера.

**Действие:** нажать «Дополнительные» → «Перейти на сайт (небезопасно)».

SSH-туннель при этом зашифрован, соединение защищено.

---

## 4. Два сервера — два туннеля

Если у вас два сервера, используйте разные локальные порты:

**Сервер RU:**
```powershell
ssh -N -T -L 33333:127.0.0.1:23432 root@<RU_IP>
```

**Сервер NL:**
```powershell
ssh -N -T -L 33334:127.0.0.1:9843 root@<NL_IP>
```

В браузере:
- RU панель: `https://127.0.0.1:33333/<webBasePath_RU>/`
- NL панель: `https://127.0.0.1:33334/<webBasePath_NL>/`

---

## 5. Закрытие порта в файрволе

После привязки к localhost рекомендуется закрыть порт панели в файрволе:

```bash
ufw deny <порт_панели>/tcp
```

Это дополнительный уровень защиты на случай, если привязка к 127.0.0.1 будет случайно снята.

---

## 6. Диагностика проблем

### Туннель не работает

**Проверить, слушает ли порт на сервере:**
```bash
ss -lntp | grep <порт_панели>
```

**Проверить локальный порт на Windows:**
```powershell
netstat -ano | findstr :<локальный_порт>
```

Должен показать `LISTENING`.

**Проверить, нет ли ошибок при создании туннеля:**
Запустить команду без `-N -T`:
```powershell
ssh -L 33333:127.0.0.1:23432 root@<IP>
```
Если появляется shell — туннель работает.

### Панель не открывается в браузере

1. Проверить, что используется **https**, а не http
2. Проверить правильность webBasePath
3. Принять предупреждение сертификата
4. Попробовать сначала без пути: `https://127.0.0.1:33333/`

### Порт занят

Если локальный порт занят:
```powershell
netstat -ano | findstr :33333
```

Использовать другой порт:
```powershell
ssh -N -T -L 33335:127.0.0.1:23432 root@<IP>
```

---

## 7. Отключение туннеля

Закрыть окно PowerShell или нажать `Ctrl+C` в окне с туннелем.

---

## 8. SSH Config для удобства

Добавить в `C:\Users\<user>\.ssh\config`:

```
Host tunnel-ru
    HostName <RU_IP>
    User root
    IdentityFile C:\Users\<user>\.ssh\id_mykey
    LocalForward 33333 127.0.0.1:23432
    SessionType none
    ExitOnForwardFailure yes

Host tunnel-nl
    HostName <NL_IP>
    User root
    IdentityFile C:\Users\<user>\.ssh\id_mykey
    LocalForward 33334 127.0.0.1:9843
    SessionType none
    ExitOnForwardFailure yes
```

Теперь туннель запускается просто:
```powershell
ssh -N tunnel-ru
ssh -N tunnel-nl
```

---

*Все IP-адреса и порты заменены на плейсхолдеры.*
