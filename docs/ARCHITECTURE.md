## Компоненты

### RU-сервер

| Компонент | Порт | Назначение |
|-----------|------|------------|
| Xray (VLESS Reality) | 443 (внешний) | Приём клиентских подключений |
| Маршрутизация | — | RU-трафик → direct, World → NL |
| Nginx | 2096 (внешний) | HTTPS-сайт + прокси подписок |
| Subscription Service | 2097 (внутренний, 127.0.0.1) | Генерация конфигов подписки |
| 3x-ui Panel | 23432 (внутренний, 127.0.0.1) | Панель управления |
| SSH | 22 | Управление сервером (только по ключу) |

### NL-сервер

| Компонент | Порт | Назначение |
|-----------|------|------------|
| Xray (VLESS Reality) | 443 (внешний) | Приём каскадного трафика от RU |
| Outbound direct | — | Выпуск трафика в интернет |
| 3x-ui Panel | 9843 (внутренний, 127.0.0.1) | Панель управления |
| SSH | 22 | Управление сервером (только по ключу) |

## Потоки трафика

### Клиент → RU-сервер

1. Клиент подключается к `RU_IP:443` по VLESS TCP Reality
2. Xray расшифровывает трафик
3. Применяется маршрутизация (см. ниже)
4. Ответ возвращается клиенту

### Каскад RU → NL

1. Трафик, не попавший под правила RU (direct/block), уходит в outbound на NL
2. Соединение: VLESS TCP Reality на `NL_IP:443`
3. NL-сервер принимает трафик и выпускает его через direct/freedom
4. Ответ возвращается: интернет → NL → RU → клиент

### Подписка

1. Клиентское приложение делает HTTPS-запрос к `https://RU_IP:2096/sub/<subId>`
2. Nginx принимает запрос, проверяет TLS
3. Проксирует запрос на `http://127.0.0.1:2097/sub/<subId>`
4. Subscription Service (внутри x-ui) генерирует конфиг
5. Ответ возвращается: x-ui → Nginx → клиент (по HTTPS)

### Панели управления

1. Администратор создаёт SSH-туннель:
   - `ssh -L 33333:127.0.0.1:23432 root@RU_IP`
   - `ssh -L 33334:127.0.0.1:9843 root@NL_IP`
2. Открывает в браузере `https://127.0.0.1:33333/webBasePath/`
3. Трафик идёт по зашифрованному SSH-туннелю
4. Панель недоступна из интернета напрямую

## Маршрутизация на RU-сервере

```

Входящий трафик
│
├── geoip:ru → direct (российские IP)
├── domain:ru → direct (российские домены)
├── private/loopback → blocked
├── bittorrent → blocked
└── всё остальное → NL-сервер (каскад)
```

## Безопасность

| Уровень | Защита |
|---------|--------|
| SSH | Только ключи Ed25519, пароль отключён |
| Панели | Только через SSH-туннель, привязаны к 127.0.0.1 |
| Подписка | HTTPS (Nginx + Let's Encrypt), нестандартный URI |
| Файрвол | UFW, открыты только необходимые порты |
| Xray | Reality (маскировка под обычный TLS) |
| Базы данных | Регулярная проверка integrity, бэкапы |

## Зависимости сервисов

```

x-ui.service
├── Xray (основной прокси + каскад)
├── Панель 3x-ui (веб-интерфейс)
└── Subscription Service (подписки)

nginx.service
└── Проксирование /sub/* → Subscription Service

certbot.timer
└── Автоматическое продление SSL-сертификатов
```

## Схема портов

```

Интернет
│
├── :22 (SSH) — доступ управление
├── :443 (Xray) — клиентские подключения
└── :2096 (Nginx HTTPS) — сайт + подписки

Внутренние (127.0.0.1):
├── :23432 — панель RU
├── :9843 — панель NL
└── :2097 — сервис подписок
```
