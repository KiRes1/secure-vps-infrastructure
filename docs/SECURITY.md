## 1. Что сделано для безопасности

### SSH

| Мера | Статус | Эффект |
|------|--------|--------|
| Вход только по ключам Ed25519 | ✅ Выполнено | Исключён брутфорс паролей |
| Passphrase на ключе | ✅ Выполнено | Второй фактор защиты ключа |
| Отключён UsePAM | ✅ Выполнено | Нет обхода через PAM |
| Отключён ChallengeResponse | ✅ Выполнено | Нет интерактивных методов входа |
| Отключён KbdInteractive | ✅ Выполнено | Нет клавиатурного ввода |
| Cloud-init переопределён | ✅ Выполнено | Файлы 50-cloud-init.conf исправлены |

### Панели управления

| Мера | Статус | Эффект |
|------|--------|--------|
| Привязка к 127.0.0.1 | ✅ Выполнено | Панели недоступны из интернета |
| SSH-туннель | ✅ Выполнено | Доступ только через зашифрованный канал |
| Нестандартный webBasePath | ✅ Выполнено | Невозможно угадать URL |
| Порты панелей закрыты в UFW | ✅ Выполнено | Дополнительный уровень защиты |

### Подписка

| Мера | Статус | Эффект |
|------|--------|--------|
| Нестандартный URI | ✅ Выполнено | Не `/sub/`, сложнее обнаружить |
| Reverse Proxy (Nginx) | ✅ Выполнено | Прямой порт подписки скрыт |
| HTTPS (Let's Encrypt) | ✅ Выполнено | Шифрование трафика подписки |
| Порт 2097 только localhost | ✅ Выполнено | Внутренний сервис не exposed |

### Файрвол

| Мера | Статус | Эффект |
|------|--------|--------|
| UFW включён | ✅ Выполнено | Фильтрация трафика |
| Открыты только нужные порты | ✅ Выполнено | Минимальная поверхность атаки |

## 2. Текущие риски

### 🔴 Высокий риск

| Риск | Описание | Что делать |
|------|----------|------------|
| Один ключ на оба сервера | Если ключ скомпрометирован — доступ ко всем серверам | Использовать разные ключи для RU и NL |
| Нет 2FA на панелях | Пароль панели — единственный фактор | Включить 2FA в настройках панели |
| Пароли панелей в браузере | Могут быть украдены через вредоносное ПО | Использовать менеджер паролей |

### 🟡 Средний риск

| Риск | Описание | Что делать |
|------|----------|------------|
| Подписка доступна извне | Хотя URI скрыт, порт 2096 открыт | Мониторить логи Nginx |
| Fail2Ban не настроен | Нет защиты от перебора SSH (хотя пароль отключён) | Установить Fail2Ban |
| Нет rate limiting | DDoS на порт 443/2096 | Настроить лимиты в Nginx/Xray |
| Самоподписанный сертификат панели | Браузер ругается | Смириться (туннель шифрует) |

### 🟢 Низкий риск

| Риск | Описание |
|------|----------|
| Перебор SubId | SubId длинные и уникальные, практически невозможно |
| Сканирование портов | Открыты только 22, 443, 2096 |
| Компрометация панели | Панель за SSH-туннелем, нужен доступ к компьютеру админа |

## 3. Рекомендации по усилению

### Что можно сделать дополнительно:

| Рекомендация | Приоритет | Сложность |
|--------------|-----------|-----------|
| Установить Fail2Ban | 🔴 Высокий | Низкая |
| Включить 2FA на панелях | 🔴 Высокий | Низкая |
| Разные SSH-ключи для RU и NL | 🟡 Средний | Низкая |
| Rate limiting в Nginx | 🟡 Средний | Средняя |
| Мониторинг логов (logwatch) | 🟢 Низкий | Низкая |
| Автоматические обновления безопасности | 🟢 Низкий | Низкая |
| Разные пароли панелей RU и NL | 🟡 Средний | Низкая |

### Установка Fail2Ban:

```bash
apt install fail2ban -y
systemctl enable fail2ban
systemctl start fail2ban
fail2ban-client status
```

### Unattended-upgrades (автообновления безопасности):

```bash
apt install unattended-upgrades -y
dpkg-reconfigure -plow unattended-upgrades
```

## 4. Аудит безопасности

### Команды для регулярной проверки:

```bash
# Открытые порты
ss -lntup

# Файрвол
ufw status numbered

# SSH ключи
cat /root/.ssh/authorized_keys

# Статус сервисов
systemctl status x-ui nginx sshd --no-pager

# Неиспользуемые пакеты
apt autoremove --dry-run

# Обновления безопасности
apt list --upgradable | grep -i security
```

### Чек-лист аудита безопасности:

- [ ] SSH — только ключи, пароль отключён
- [ ] Панели — только 127.0.0.1
- [ ] Подписка — HTTPS, нестандартный URI
- [ ] Файрвол — только нужные порты
- [ ] Сертификаты — не истекли
- [ ] База данных — integrity_check OK
- [ ] Бэкапы — актуальные
- [ ] Обновления — установлены
- [ ] Логи — без подозрительной активности

## 5. Действия при компрометации

### Если есть подозрение на взлом:

1. **Изолировать:** отключить сервер от сети (через панель VPS)
2. **Снапшот:** сделать снапшот для последующего анализа
3. **Сменить все ключи и пароли:**
   - SSH-ключи
   - Пароли панелей
   - Reality keys (потребует перевыпуска клиентов)
   - UUID клиентов
4. **Проверить логи:**
   ```bash
   journalctl -u x-ui -n 1000
   journalctl -u nginx -n 1000
   cat /var/log/auth.log | grep Accepted
   ```
5. **Восстановить из чистого бэкапа** (если есть уверенность, что он не затронут)
6. **Усилить защиту** по рекомендациям выше
