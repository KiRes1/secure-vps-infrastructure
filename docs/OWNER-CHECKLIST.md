## 1. Доступ к серверам

- [ ] Владелец может войти по SSH на **RU-сервер** своим ключом:
  ```powershell
  ssh -i <путь_к_ключу> root@<RU_IP>
  ```

- [ ] Владелец может войти по SSH на **NL-сервер** своим ключом:
  ```powershell
  ssh -i <путь_к_ключу> root@<NL_IP>
  ```

- [ ] Парольная аутентификация SSH отключена на **обоих** серверах

- [ ] Настроен SSH Config для удобства:
  ```
  Host ru-server
      HostName <RU_IP>
      User root
      IdentityFile <путь_к_ключу>
      IdentitiesOnly yes

  Host nl-server
      HostName <NL_IP>
      User root
      IdentityFile <путь_к_ключу>
      IdentitiesOnly yes
  ```

- [ ] Владелец знает passphrase от своего SSH-ключа

## 2. Панели управления

- [ ] Владелец знает порты панелей:
  - RU: `23432`
  - NL: `9843`

- [ ] Владелец знает webBasePath:
  - RU: `<уникальный_путь_RU>`
  - NL: `<уникальный_путь_NL>`

- [ ] Владелец умеет запускать SSH-туннели для доступа к панелям:
  ```powershell
  ssh -N -T -L 33333:127.0.0.1:23432 root@<RU_IP>
  ssh -N -T -L 33334:127.0.0.1:9843 root@<NL_IP>
  ```

- [ ] Владелец вошёл в **RU-панель** через туннель

- [ ] Владелец вошёл в **NL-панель** через туннель

- [ ] Логины и пароли панелей переданы через защищённый канал

- [ ] Владелец знает, где хранятся логины/пароли (менеджер паролей)

## 3. Клиенты и подписки

- [ ] Владелец знает, что клиенты создаются **только на RU-сервере**

- [ ] Владелец создал тестового клиента на RU

- [ ] Владелец получил ссылку подписки тестового клиента

- [ ] Владелец проверил подписку в браузере (HTTP 200)

- [ ] Владелец импортировал подписку в VPN-приложение

- [ ] Подключение работает, IP определяется правильно

- [ ] Владелец знает формат ссылки подписки:
  ```
  https://<RU_IP>:2096/<URI_путь>/<subId>
  ```

- [ ] Владелец знает, что на NL-сервере клиентов создавать **не нужно**

- [ ] Владелец умеет:
  - Создавать клиента
  - Отключать/включать клиента
  - Менять лимиты и срок
  - Сбрасывать трафик
  - Удалять клиента
  - Менять UUID для отзыва доступа

## 4. Маршрутизация

- [ ] Владелец проверил, что `2ip.ru` показывает **RU IP**

- [ ] Владелец проверил, что `ifconfig.me` показывает **NL IP**

- [ ] Российские сайты открываются (yandex.ru, vk.com, gosuslugi.ru)

- [ ] Иностранные сайты открываются (google.com, github.com)

- [ ] Владелец понимает схему каскада RU → NL

- [ ] Владелец знает, что на NL входящий трафик только от RU-сервера

## 5. Бэкапы

- [ ] Владелец знает расположение бэкапов:
  - RU: `/root/backup-.../`
  - NL: `/root/backup-.../`

- [ ] Владелец знает команды восстановления из бэкапа

- [ ] Владелец умеет создавать снапшоты VPS у провайдера

- [ ] Владелец понимает, что перед изменениями нужно:
  1. Сделать снапшот
  2. Сделать локальный бэкап
  3. Проверить integrity базы

## 6. Безопасность

- [ ] Владелец понимает, какие порты открыты и почему

- [ ] Владелец знает, что панели **недоступны** из интернета (только через туннель)

- [ ] Владелец знает, что подписка доступна только по HTTPS через Nginx

- [ ] Владелец понимает риски (см. SECURITY.md)

- [ ] Владелец знает, что стандартный путь `/sub/` заменён на уникальный

- [ ] Принято решение по 2FA на панелях: да / нет

- [ ] Принято решение по Fail2Ban: установить / не устанавливать

## 9. Финальное решение по доступам

- [ ] Принято решение по временному ключу помощника:
  - [ ] Оставить (помощник сохраняет доступ)
  - [ ] Удалить (помощник теряет доступ)

- [ ] Если удалить — выполнить на обоих серверах:
  ```bash
  nano /root/.ssh/authorized_keys
  # удалить строку с ключом помощника
  ```
