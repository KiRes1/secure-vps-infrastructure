# 👤 Создание нового клиента

Пошаговая инструкция добавления пользователя VPN.

## Важное правило

**Новый клиент создаётся ТОЛЬКО на RU-сервере.**

На NL-сервере создавать клиентов **не нужно**. Там только служебный клиент для каскада RU → NL.

---

## Пошаговая инструкция

### Шаг 1: Зайти в панель RU-сервера

Запустить SSH-туннель (если панель скрыта):

```powershell
ssh -N -T -L 33333:127.0.0.1:23432 root@<RU_IP>
```

Открыть в браузере:

```
https://127.0.0.1:33333/<webBasePath>/
```

Войти с логином и паролем администратора.

---

### Шаг 2: Открыть рабочий Inbound

1. В боковом меню выбрать **Входящие** (Inbounds)
2. Найти рабочий VLESS TCP Reality inbound на порту 443
3. Нажать на него для редактирования

---

### Шаг 3: Добавить клиента

1. Перейти в раздел **Клиенты** (Clients)
2. Нажать **Добавить клиента** (Add Client) или **+**

Заполнить поля:

| Поле | Значение | Примечание |
|------|----------|------------|
| **Email** | Понятное имя клиента | Например: `Ivanov-Moscow` |
| **UUID** | Сгенерировать новый | Кнопка Generate, **не копировать от другого клиента** |
| **Flow** | `xtls-rprx-vision` | Обязательно для Reality |
| **Лимит IP** | `0` (безлимит) или число | Сколько устройств одновременно |
| **Лимит трафика (GB)** | `0` (безлимит) или число | Общий лимит в гигабайтах |
| **Срок действия** | `0` (бессрочно) или дата | Когда истечёт доступ |
| **Sub ID** | Сгенерировать новый | Уникальный ID для ссылки подписки |
| **Включить** | ✅ Да | Если не включить — клиент не сможет подключиться |

4. Нажать **Сохранить** (Save)

---

### Шаг 4: Получить ссылку подписки

1. В списке клиентов найти созданного
2. Нажать на него → **Копировать ссылку** (в блоке «Информация о подписке»)

**Формат ссылки подписки:**

```
https://<RU_IP>:2096/<URI_путь>/<subId_клиента>
```

**Формат прямой ссылки (vless://):**

```
vless://<UUID>@<RU_IP>:443?encryption=none&flow=xtls-rprx-vision&fp=firefox&pbk=<pub_key>&security=reality&sid=<short_id>&sni=<sni>&type=tcp#<имя_клиента>
```

---

### Шаг 5: Проверить ссылку

Перед отправкой клиенту проверить:

1. Ссылка содержит внешний IP сервера (не `localhost`, не `127.0.0.1`)
2. Ссылка открывается в браузере (должна скачать конфиг или показать текст)
3. Импортировать в клиентское приложение и проверить подключение

---

### Шаг 6: Передать клиенту

Передать **только ссылку подписки** или **прямую ссылку**.

**Не передавать:**
- UUID других клиентов
- Пароли панели
- SSH-ключи
- Внутренние адреса и порты

---

## Как клиент подключается

1. Установить приложение:
   - Android: v2rayNG, Nekobox, Happ
   - iOS: Streisand, Foxray
   - Windows: v2rayN

2. Импортировать подписку:
   - **Подписка:** вставить URL в поле Subscription
   - **Прямая ссылка:** импортировать конфиг или отсканировать QR

3. Если пинг `n/a`:
   - Проверить DNS на устройстве (должен быть чистый: `1.1.1.1`)
   - Проверить, что в конфиге внешний IP, а не `localhost`

---

## Что делать нельзя

- ❌ Создавать клиента на NL-сервере
- ❌ Копировать UUID от другого клиента
- ❌ Использовать одинаковый SubId для разных клиентов
- ❌ Передавать `localhost`-ссылки клиентам
- ❌ Менять параметры Reality (public key, shortId) без перевыпуска всех клиентов

---

## Проверка после создания

```bash
# Проверить подписку на сервере
curl -I https://<RU_IP>:2096/<URI_путь>/<subId>

# Должен быть HTTP 200
```

---

*Все IP, порты и идентификаторы заменены на плейсхолдеры.*
